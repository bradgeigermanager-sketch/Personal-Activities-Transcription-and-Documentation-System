PATDS

# 🟦 Component #5 — NLP Enrichment Service  
This is the **intelligence core** of PATDS. It takes the diarized transcript and produces structured meaning:

- Summaries  
- Action items  
- Topics  
- Entities  
- Sentiment (optional)  
- Tags  
- Key decisions  
- Follow‑ups  
- Highlights  

This is what turns raw text into a **personal knowledge system**.

---

# 🧩 Subcomponents

## 1. **Summarization Engine**  
### Purpose  
Create concise, readable summaries of conversations or monologues.

### MVP  
- Use a small‑to‑medium transformer (T5‑base, Llama‑3‑8B, etc.)  
- Input: diarized transcript  
- Output:  
  ```json
  {
    "summary": "Discussed sending the proposal, scheduling a meeting with Alex, and planning next week's tasks."
  }
  ```

### Advanced  
- Multi‑level summaries:  
  - Short (1–2 sentences)  
  - Medium (paragraph)  
  - Detailed (bullet points)  

---

## 2. **Action Item Extraction**  
### Purpose  
Detect tasks, commitments, deadlines, and follow‑ups.

### MVP  
- Regex + LLM hybrid:  
  - Look for verbs like “send”, “schedule”, “remind”, “follow up”.  
  - LLM confirms and structures them.

### Output  
```json
{
  "actions": [
    {
      "description": "Send the proposal",
      "due": "tomorrow",
      "source_segment": 0
    },
    {
      "description": "Schedule meeting with Alex",
      "due": "next week",
      "source_segment": 1
    }
  ]
}
```

---

## 3. **Entity Extraction**  
### Purpose  
Identify people, places, organizations, dates, and topics.

### MVP  
- spaCy NER or a small transformer  
- Entities stored as:  
  ```json
  {
    "entities": [
      { "type": "PERSON", "value": "Alex" },
      { "type": "DATE", "value": "tomorrow" }
    ]
  }
  ```

---

## 4. **Topic Tagging**  
### Purpose  
Automatically categorize content.

### MVP  
- Predefined topic list:  
  - work, health, finance, relationships, errands, planning  
- LLM assigns tags.

### Output  
```json
{
  "topics": ["work", "planning"]
}
```

---

## 5. **Sentiment & Tone Analysis (Optional)**  
Useful for journaling, mood tracking, and insights.

### Output  
```json
{
  "sentiment": "neutral",
  "tone": "focused"
}
```

---

## 6. **Highlight Extraction**  
### Purpose  
Pull out the most important lines or moments.

### MVP  
- LLM selects 1–3 key sentences.

---

## 7. **NLP Pipeline Orchestrator**  
### Flow  
1. Receive `"diarization_complete"` event  
2. Fetch transcript  
3. Run:  
   - Summarization  
   - Action extraction  
   - Entity extraction  
   - Topic tagging  
   - Sentiment (optional)  
4. Combine into enriched transcript  
5. Save to Processed Data Store  
6. Emit `"nlp_complete"` event  

---

# 🧱 Implementation Blueprint (MVP)

### Backend  
- Python service (FastAPI)  
- Models:  
  - T5‑base for summarization  
  - spaCy for NER  
  - Small LLM for action extraction  

### Data Model  
```json
{
  "transcript_id": "uuid",
  "summary": "...",
  "actions": [...],
  "entities": [...],
  "topics": [...],
  "sentiment": "neutral"
}
```

### Worker  
- Runs asynchronously  
- Subscribed to diarization events  

### Storage  
- Writes to `processed_transcripts` collection
