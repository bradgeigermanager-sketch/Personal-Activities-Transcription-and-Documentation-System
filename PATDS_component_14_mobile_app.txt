# 🟦 Component #14 — Mobile App  
The Mobile App, the first module of the Interface Layer. This is where PATDS becomes something you can *carry*, *use instantly*, and *live inside of*. The mobile app is the user’s primary gateway into the system, so we design it to be fast, intuitive, and deeply integrated with the pipeline.
The PATDS mobile app has three core responsibilities:

1. **Capture** — audio, notes, tasks, quick events  
2. **Review** — timeline, documents, tasks, insights  
3. **Control** — trigger summaries, search, voice commands  

Everything else is optional polish.

---

# 🧩 Subcomponents

## 1. **App Architecture**  
Choose a cross‑platform or native approach.

### Recommended  
- **Flutter** — best balance of speed, UI quality, and dev velocity  
- **React Native** — great if your backend is JS/TS  
- **Native Swift/Kotlin** — best performance, more work  

### Core modules  
- Audio recorder  
- Timeline viewer  
- Task manager  
- Document viewer  
- Search interface  
- Settings & privacy  

---

## 2. **Audio Capture Module**  
This connects directly to Component #1.

### Features  
- Start/stop recording  
- Waveform visualization  
- Background recording  
- Auto‑upload on stop  
- Offline mode (queue uploads)  

### Implementation  
- Flutter: `flutter_sound`  
- React Native: `react-native-audio-recorder-player`  
- Native: AVAudioRecorder / MediaRecorder
